ただしその際、コアだけで依存などが完結するように調整をしてください。
基本的にコアのソースは依存がコアだけで完結するように調整するだけで他は変えないで移してください。
移すときに、それぞれのcファイルにインクルードを書き込むことを忘れないでください。
コアだけで完結するようにしてください。

例えばエミュとして不要なデバッグ系は分離してオプションで切り替えられるようにして、別管理。
通常は無効化。
そして、gbaコアに必要なものはエミュ機能と、精度補助だけで、フロントエンド？はコア単体には不要。
なので、コアからフロントエンドやデバッグ系などは分離して削除または隔離。
ファイル選択などのos別の抽象化などは不要。
今回はiosとlinuxでのみの使用を仮定している。
linuxでは、romやbiosは、コマンドで直接指定し、iosでは、ファイル選択で指定する形で。
フロントエンド依存は削除で。
コアがエミュコアとして完結するように大幅に組み替える。
